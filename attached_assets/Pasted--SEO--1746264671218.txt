대시보드 개요
대시보드는 스마트스토어 SEO를 위한 분석 도구로, 유저가 저장한 쿼리 분석 정보를 시각적으로 제공하고, 유저가 다시 로그인할 때마다 변화된 분석 결과를 확인할 수 있는 대시보드를 만들고자 합니다. 이 대시보드에서는 유저의 쿼리 분석 결과를 보여주고, 변화된 값을 추적할 수 있게 됩니다.

주요 요구 사항
로그인 후 쿼리 분석 정보 갱신 및 변화 추적:

유저가 로그인할 때마다, 기존에 저장된 쿼리를 크롬 확장프로그램에서 다시 분석하여 변화된 부분을 추적하고 대시보드에 표시합니다.

유저가 새로운 쿼리를 추가하면, 크롬 확장프로그램에서 해당 쿼리를 분석하고 Firebase 데이터베이스에 저장한 후, 변화된 부분을 대시보드에 반영합니다.

데이터 구조:
데이터는 다음과 같은 구조로 Firebase Firestore에 저장됩니다.

json
복사
편집
analysisLogs/
  └── {email}/
      └── {query}/
          ├── lastUpdated: YYYY-MM-DD
          ├── keywords: [{ key: string, value: number }]
          ├── keywordCounts: [{ key: string, value: number }]
          ├── tags: [{ key: string, value: number }]
          └── savedAt: ISO timestamp
쿼리 저장 제한:

각 유저는 최대 3개의 쿼리만 저장할 수 있습니다.

쿼리가 이미 저장된 경우, 새로운 쿼리는 저장되지 않도록 하며, 기존 쿼리는 변경이 가능합니다.

쿼리의 날짜가 다를 경우에는 새로운 날짜 필드가 생성되지만, 같은 날짜에는 기존 쿼리로 덮어쓰지 않도록 합니다.

변화 추적 기능:

대시보드에서 저장된 키워드, 키워드 카운트, 태그의 key와 value 변화가 발생한 경우 이를 시각적으로 강조합니다.

변화가 발생한 데이터는 추가된 항목, 변경된 항목, 삭제된 항목으로 구분하여 표시합니다.

예시:

추가된 항목: + 기호와 함께 초록색으로 강조

변경된 항목: ↗나 ↘ 기호로 변화량을 파란색으로 표시

삭제된 항목: - 기호와 함께 빨간색으로 표시

쿼리 삭제 및 추가 기능:

유저는 기존에 저장된 쿼리를 삭제하거나 새로운 쿼리를 추가할 수 있습니다.

쿼리 삭제 시 Firebase 데이터베이스에서 해당 쿼리 삭제.

쿼리 추가 시 크롬 확장프로그램에서 분석 후 새로운 데이터를 Firestore에 저장하고 대시보드에 반영됩니다.

대시보드 UI 설계
쿼리 목록:

유저가 저장한 최대 3개의 쿼리를 대시보드에 나열합니다.

각 쿼리에 대해 변화된 데이터를 표시하며, 변경된 키워드, 태그, 키워드 카운트를 항목별로 분리해서 보여줍니다.

변화된 값 표시:

변화가 발생한 키워드, 키워드 카운트, 태그에 대해 추가된 항목, 변경된 항목, 삭제된 항목을 색상과 기호로 시각화하여 유저가 쉽게 변화된 데이터를 파악할 수 있도록 합니다.

변화량을 수치적으로 표시하고, 변경된 값을 강조하여 표시합니다.

쿼리 분석 기능:

유저가 추가한 쿼리에 대해 크롬 확장프로그램에서 분석 후, 변화된 부분을 대시보드에 실시간으로 반영합니다.

삭제 및 추가:

유저는 대시보드에서 쿼리를 삭제하거나 새로운 쿼리를 추가할 수 있는 버튼을 제공합니다.

쿼리 삭제 시, Firestore에서 해당 쿼리 삭제 후 대시보드에서 반영합니다.

쿼리 추가 시, 크롬 확장프로그램에서 분석 후 새로운 쿼리를 Firebase에 저장하고, 변화된 정보를 대시보드에 반영합니다.

전체 흐름 정리
유저 로그인:

유저가 로그인하면, 기존에 저장된 쿼리들을 가져옵니다.

그 후, 크롬 확장프로그램에서 해당 쿼리들을 분석하고, 변화된 부분을 데이터베이스에 반영한 후, 대시보드에 표시합니다.

쿼리 추가/삭제:

유저가 새로운 쿼리를 추가하면, 해당 쿼리를 크롬 확장프로그램에서 분석하고, 결과를 Firebase에 저장합니다.

유저가 기존 쿼리를 삭제하면, Firebase에서 해당 쿼리 삭제 후 대시보드에서 반영합니다.

변화 추적:

대시보드는 저장된 쿼리와 변화된 데이터를 시각적으로 표시하고, 변경된 값과 추가된 값을 쉽게 파악할 수 있게 합니다.

이런 방식으로 대시보드가 설계되며, 유저는 자신의 쿼리와 관련된 변화된 정보를 실시간으로 확인할 수 있습니다.
